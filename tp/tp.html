Création d’un nouveau projet

Dans le TP Création d’un environnement de développement vous avez découvert l’arborescence d’un projet web de base avec la possibilité de mettre en place des tests unitaires. Nous allons repartir de cette arborescence pour ce TP.

Dans ce TP nous allons utiliser une nouvelle fois le mode de développement TDD (Test-driven development). Cela signifie que nous allons définir un objectif à atteindre et développer le code qui permet de l’atteindre.

Pour vous initier à l’utilisation d’un projet déjà existant disponible dans un dépôt git et que vous devez améliorer, vous allez faire un fork (divergence).

Le projet à récupérer est celon votre groupe :

  • Groupe A : TP Objet PHP grp A
  • Groupe B : TP Objet PHP grp B
  • Groupe C : TP Objet PHP grp C

Quand vous aurez terminé le TP, vous proposerez une “Merge Request” vers le projet de départ. Le développeur propose son code à l’aide de cette technique à un chef de projet qui valide les propositions ou demande des modifications. Ce mode de fonctionnement est proche de celui que vous rencontrerez dans les entreprises.

Nous allons créer un nouveau projet tp-objet-php dans lequel nous allons mettre en oeuvre la syntaxe PHP.

  1. Créez une divergence du projet TP Objet PHP disponible sur gitlab.
  2. Clonez le projet dans votre environnement de travail.
  3. Récupérez les dépendances du projet.

Attention

Il faut prendre l’habitude de suivre cette consigne :

1 question = 1 commit

    git add .
    git commit -am "réponse à la question XXX"

Et faire une sauvegarde dans votre dépôt gitlab à la fin de la séance.

    git push origin main

Création de la classe Point

Nous allons créer une classe Point qui a deux propriétés $x et $y de type float. Ces propriétés sont protected.

Attention

Nous allons suivre la règle utilisée en java, qui place le code d’une seule classe publique dans un fichier. Le nom du fichier correspond au nom de la classe. Cette règle permet d’utiliser un mécanisme de recherche et de chargement automatique des classes de notre application qui est utilisé dans le script vendor/autoload.php généré par la commande composer.

Dans votre fichier composer.json, vous devez indiquer un nom de package dans le fichier. Si vous avez indiqué TpWeb/ObjetPhp alors vous devez ajouter la ligne :

namespace TpWeb\ObjetPhp;

Cela donne à la création de la classe Point dans le fichier src/Point.php de votre projet :

<?php
/**
 * Contenu du fichier Point.php
 */

namespace TpWeb\ObjetPhp;

class Point {

}

    Point ( 0.00,  0.00) <-- les coordonnées x et y sont affichés à l'aide de 5 caractères (%5.2f)

Attention

Un attribut de classe est un attribut déclaré statique. Un attribut de classe existe dans la classe, et il est accessible par les instances de la classe dans laquelle il est déclaré. Cela signifie que l’attribut n’existe qu’en un seul exemplaire quel que soit le nombre d’instances. Si l’attribut vaut 18 par exemple, l’ensemble des instances de la classe récupérera cette valeur.

class Foo {
   static $numEmp = 18;

   // utilisation de l'attribut dans une fonction de la classe
   function foo() {
     // ...
     return Foo::$numEmp + 10;
     // ou return self::$numEmp + 10;
   }
}
